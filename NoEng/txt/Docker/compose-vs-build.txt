## Compose vs build 

### Назначение:

Docker Compose: Управление многоконтейнерными приложениями, их запуск, масштабирование и оркестрация.
docker build: Создание Docker образов из Dockerfile.
docker run: Используется для запуска и управления одиночными контейнерами. Это низкоуровневый инструмент, требующий явного указания всех параметров при каждом запуске.

### Использование:

Docker Compose: Используется для запуска и управления несколькими сервисами, которые могут зависеть друг от друга.
docker build: Используется для создания образов, которые затем могут быть запущены как отдельные контейнеры или использованы в Docker Compose.
docker run: Подходит для быстрого запуска отдельных контейнеров или выполнения одноразовых задач.

### Файлы конфигурации:

Docker Compose: Использует файл docker-compose.yml для определения конфигурации сервисов.
docker build: Использует файл Dockerfile для определения шагов создания образа.
docker run: Параметры конфигурации указываются непосредственно в командной строке.

### Как они работают вместе:

Создание образов с помощью docker build:

Сначала вы создаете Dockerfile для каждого компонента вашего приложения и создаете образы с помощью команды docker build.

Запуск приложений с помощью Docker Compose:

Затем вы используете Docker Compose для оркестрации этих образов и управления многоконтейнерным приложением, определяя все сервисы и их конфигурации в docker-compose.yml.
