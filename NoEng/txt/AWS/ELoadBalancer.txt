## ELB 

Elastic Load Balancer (ELB) является ключевым компонентом инфраструктуры Amazon Web Services (AWS), предназначенным для автоматического распределения входящего трафика приложений между несколькими экземплярами Amazon EC2. Это обеспечивает высокую доступность, отказоустойчивость и масштабируемость приложений. Рассмотрим основные аспекты и функции ELB:

Основные типы ELB

### Application Load Balancer (ALB):

Предназначен для приложений, работающих на 7 уровне (Layer 7) модели OSI, что позволяет делать маршрутизацию на основе содержимого HTTP/HTTPS-запросов.
Поддерживает функции маршрутизации на основе URL, заголовков, методов HTTP и параметров запроса.
Поддерживает веб-сокеты и HTTP/2.

### Network Load Balancer (NLB):

Работает на 4 уровне (Layer 4) модели OSI и предназначен для высокопроизводительных приложений, требующих низкой задержки.
Способен обрабатывать миллионы запросов в секунду и сохранять очень низкую задержку.
Поддерживает TCP и UDP.

### Gateway Load Balancer (GWLB):

Специально разработан для интеграции с сетевыми виртуальными устройствами, такими как фаерволлы и системы обнаружения вторжений.
Объединяет функции балансировки нагрузки и маршрутизации для сетевых устройств.

### Classic Load Balancer (CLB):

Работает на уровнях 4 и 7 модели OSI.
Поддерживает как HTTP/HTTPS, так и TCP-трафик.
Обычно используется для приложений, созданных на более ранних версиях AWS, хотя все новые разработки обычно используют ALB или NLB.
Основные функции ELB

### Автоматическое распределение нагрузки:

ELB автоматически распределяет входящий трафик между экземплярами EC2, контейнерами или другими целевыми ресурсами в одной или нескольких зонах доступности (Availability Zones).

### Высокая доступность и отказоустойчивость:

ELB распределяет трафик через несколько зон доступности, обеспечивая непрерывность работы приложения даже при сбое одной из зон.

### Здоровье экземпляров:

ELB регулярно проверяет состояние целевых ресурсов (например, экземпляров EC2) и направляет трафик только на здоровые ресурсы.

### Масштабируемость:

ELB автоматически масштабируется в ответ на изменения трафика, обеспечивая оптимальную производительность приложения.

### Безопасность:

Поддержка SSL/TLS шифрования для безопасного обмена данными.
Интеграция с AWS Certificate Manager (ACM) для управления сертификатами.
Настройка групп безопасности и политики доступа.

### Гибкая маршрутизация:

ALB позволяет делать интеллектуальную маршрутизацию на основе содержимого HTTP/HTTPS-запросов.
NLB обеспечивает статический IP-адрес и возможность использования собственных IP-адресов.

### Примеры использования
- Веб-приложения:

Обеспечение высокой доступности и отказоустойчивости для веб-сайтов и API.
Маршрутизация трафика на основе URL и других параметров.
Мобильные и интернет-приложения:

Масштабирование и балансировка трафика для мобильных приложений и интернет-сервисов.
Микросервисные архитектуры:

ALB хорошо подходит для микросервисных архитектур, позволяя маршрутизировать запросы к различным службам в зависимости от содержимого запроса.
Корпоративные приложения:

Поддержка высокопроизводительных и низколатентных приложений с помощью NLB.
Управление и мониторинг
AWS Management Console: Веб-интерфейс для создания и управления ELB.
AWS CLI: Командная строка для автоматизации управления ELB.
AWS SDKs: Библиотеки для взаимодействия с API AWS на различных языках программирования.
Amazon CloudWatch: Мониторинг метрик и журналов для ELB, таких как запросы, задержки, ошибки и проверки здоровья.
ELB является мощным инструментом для обеспечения высокой доступности, отказоустойчивости и масштабируемости приложений, работающих в AWS.
